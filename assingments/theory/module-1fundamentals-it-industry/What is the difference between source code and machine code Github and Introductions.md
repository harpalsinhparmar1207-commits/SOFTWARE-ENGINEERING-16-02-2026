# What is the Difference Between Source Code and Machine Code?

## Source Code
- Human-readable text written in programming languages (Python, Java, C++, etc.)
- Created and maintained by developers
- Requires compilation or interpretation to run
- Easy to understand, debug, and modify

## Machine Code
- Low-level binary instructions (1s and 0s) executed directly by the CPU
- Generated from source code via compilers or interpreters
- Cannot be easily read or modified by humans
- Optimized for processor execution

## Key Differences

| Aspect | Source Code | Machine Code |
|--------|-------------|--------------|
| **Format** | Text-based, human-readable | Binary, processor-specific |
| **Readability** | High | Low |
| **Creation** | Written by programmers | Generated by compilers |
| **Portability** | Portable across platforms | Platform-specific |
| **Execution** | Requires translation | Directly executable |

## GitHub & Version Control
Store source code in GitHub repositories to track changes, collaborate with teams, and maintain code history. Machine code artifacts are typically not version controlled.
